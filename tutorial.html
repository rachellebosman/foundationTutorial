Tutorial

Dit is een tutorial waarbij de basis elementen van het framework worden uitgelegt zodat je snel aan de slag kunt met het maken van je eigen website. Deze website is zelf ook helemaal gemaakt met behulp van het framework, alles wat je op deze pagina ziet is puur het framework zelf.

Als eerste moet je natuurlijk het framework downloaden als je dit nog niet hebt gedaan. Daarna moet je de gedownloade bestanden in je eigen project zetten. Tussen deze bestanden bevindt zich een index.html betandje. Hierin kan je alle code verwijderen tussen de <body> </body> zodat je een lege web pagina hebt. Om te testen of de webpagina werkt typt de volgende code tussen de body tags: <h1> Hello World </h1>. Als je deze tekst op je web browser hebt staan werkt de webpagina en kunnen we verder met de tutorial.

Links

Het is heel makkelijk om een menu met links te maken, Foundation geeft hiervoor verschillende opties. Dit is het standaard menu:

  • Link 1
  • Link 2

In de code zie je dat de <ul> class="menu"> er voor zorgt dat het menu gelijk een opmaak krijgt waarbij de links naast elkaar worden geplaatst. Wil je dingen anders aan het menu hoef je simpel weg alleen de class te veranderen. Als je de links onder elkaar wilt dan moet je de class veranderen naar <ul> class="vertical menu">. Ook heb je de mogelijkheid om de positie van het menu te veranderen dit kan met behulp van de align class. Dus wil je het menu bijvoorbeeld in het midden dan typ de de volgende code: <ul> class="align-center menu">. Wil je het menu aan de rechter kan gebruik dan align-right.

Buttons

In het voorbeeld van het formulier zag je ook een blauwe button zitten, Foundation heeft veel keuze uit verschillende buttons. Je hebt 2 verschillende manieren om een button te gebruiken namelijk met <a>. Deze gebruik je voor links naar andere pagina's. De basis code voor een link ziet er zo uit: <a href="#" class="button> naam link </a>. De andere button is <button>. Deze wordt gebruikt wanneer er op de pagina iets wordt veranderd in combinatie met javascript (dus bij acties). Hiervoor ziet de code er zo uit: <button type="button" class="button"> naam button <button/>.

Voor de buttons heb veel verschillende maten en kleuren, deze kan je bepalen door de button de juiste class mee te geven. Hieronder zie je de basis buttons met de kleuren (de eerste rij buttons), de code voor de eerste button is als volgt: <a class="button primary" href="#"> "primary" <button/> Door de class te veranderen van "primary" naar een van de onderstaande (zie buttons) verander je de button naar de button in die kleur. De onderste rij buttons zijn "hollow", deze kan je simpel weg gebruiken door hollow aan de class toe te voegen. Wil je dus bijvoorbeeld de eerste button uit de onderste rij dan maak je gebruik van de volgende code: <a class="hollow button secondary" href="#"> "hollow secondary" <button/>.

"primary" "secondary" "success" "alert" "warning"

Ook is het mogelijk om de grootte van de buttons te bepalen door een van de volgende tags in de class te gebruiken:

"button tiny" "button small" "button" "button large"

Responsive design

Wanneer je het scherm van de pagina kleiner maakt zie je dat alle elementen mee bewegen en dat alles er nog steeds netjes en leesbaar uit ziet. Dit komt door het grid-systeem waarmee Foundation werkt. We gaan dit uitleggen met behulp van een voorbeeld. Stel je wilt net als op de home page 3 afbeeldingen naast elkaar hebben met wat tekst en wanneer het scherm kleiner wordt deze netjes onder elkaar komen (zie onderstaande afbeeldingen, maak het scherm wat kleiner om het in actie te zien). Om dit te bereiken moeten we gebruik maken van het grid systeem.

Om te beginnen moeten we in de code noteren dat we gebruik gaan maken van het gridsysteem Dit doen we door een div aan te maken met een class
<div class="grid container"> </div> hiertussen komen de rijen. Het grid systeem bestaat dus uit rijen, deze geven wij de volgende class <div class="grid-x"> </div>. Dus alles wat tussen deze code wordt geplaatst komt in dezelfde rij terecht. Elke rij bestaat uit 12 vakken, dus zoals in het voorbeeld bij 3 afbeeldingen, bestaat elk stukje uit 4 vakken (dit weten we door de volgende simpele rekensom: 12/3=4). De code voor elk stukje (cell) is hieronder te zien. In de class geven we de grootte aan van het, in dit geval: medium & uit hoeveel vlakken het moet bestaan (4).

Wat er in het bovenstaande voorbeeld gebeurd is wanneer je het scherm klein maakt dat de elementen onder elkaar komen. Maar wat nou als je bijvoorbeeld een rij hebt van 4 elementen en op een klein scherm wil je er 2 naast elkaar. Bij het onderstaande voorbeeld als je het scherm kleiner maakt zie je dat er 2 elementen naast elkaar komen ipv alle elementen onder elkaar (probeer maar uit).
